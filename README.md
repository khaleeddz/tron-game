# لعبة الأثر (Tron) - 4 لاعبين أونلاين

لعبة جماعية تشبه اللعبة الموجودة في الفيديو: كل لاعب يتحكم بقطعة تتحرك باستمرار على شبكة
وتترك خلفها أثراً بلونه. إذا اصطدم لاعب بالحائط أو بأي أثر (حتى أثره هو) يخرج من اللعبة.
الفائز هو آخر لاعب باقٍ، أو صاحب أكبر نسبة استحواذ على اللوحة عند امتلائها.
شريط النقاط أعلى الشاشة يوضح نسبة كل لاعب لحظياً، تماماً كما في الفيديو.

## المميزات
- تدعم حتى **4 لاعبين** في نفس الغرفة.
- لاعب واحد "يستضيف" الغرفة وينشئ رابط دعوة يرسله لأصدقائه.
- كل لاعب ينضم يحصل تلقائياً على **لون عشوائي** من الألوان المتبقية.
- تحكم عبر: أزرار الاتجاهات على الشاشة (D-Pad)، أو الأسهم/WASD على الكيبورد، أو السحب باللمس (Swipe).
- اللعبة **أونلاين حقيقياً**: الخادم هو من يتحكم بمنطق اللعبة (authoritative)، وكل لاعب يفتح
  الرابط من جهازه (موبايل أو كمبيوتر) ويلعب في نفس الوقت.

---

## 1) التشغيل محلياً (على جهازك)

يتطلب [Node.js](https://nodejs.org) (نسخة 18 أو أحدث).

```bash
cd tron-game
npm install
npm start
```

بعدها افتح المتصفح على: `http://localhost:3000`

---

## 2) اللعب مع أصدقائك أونلاين (بدون استضافة دائمة)

بما أنك تريد اللعب أنت وأصدقاؤك فقط دون نشر الموقع للجميع، أسهل طريقة هي عمل **نفق (Tunnel)**
من جهازك إلى الإنترنت مؤقتاً أثناء اللعب، باستخدام **ngrok** (مجاني):

1. شغّل السيرفر محلياً كما في الخطوة السابقة (`npm start`).
2. حمّل ngrok من https://ngrok.com/download وسجّل حساب مجاني.
3. في الطرفية نفّذ:
   ```bash
   ngrok http 3000
   ```
4. سيعطيك رابطاً عاماً شبيهاً بـ `https://xxxx.ngrok-free.app` — أرسل هذا الرابط لأصدقائك.
5. أي شخص يفتح الرابط سيدخل صفحة اللعبة مباشرة، وأنت تنشئ الغرفة وترسل لهم "رابط الغرفة"
   من داخل اللعبة نفسها (زر "نسخ الرابط" في غرفة الانتظار).

ملاحظة: يجب أن يبقى جهازك وسيرفرك يعملان طوال وقت اللعب لأن اللعبة تمر عبر جهازك.

بديل آخر لنفس الفكرة: **Cloudflare Tunnel** (مجاني بدون حساب أحياناً):
```bash
cloudflared tunnel --url http://localhost:3000
```

---

## 3) جعلها متاحة أونلاين بشكل دائم (لأي شخص)

إذا أردت لاحقاً أن تكون متاحة دائماً حتى لو أغلقت جهازك، ارفعها على استضافة مجانية تدعم Node.js
و WebSockets، مثل:

- **Render.com** (الأسهل): أنشئ حساب → New → Web Service → اربطه بمستودع GitHub يحتوي هذا المجلد
  → Build Command: `npm install` → Start Command: `npm start` → احفظ. سيعطيك رابطاً دائماً مثل
  `https://your-game.onrender.com`.
- **Railway.app**: نفس الفكرة، ارفع المشروع واختر Deploy.
- **Fly.io**: يتطلب تثبيت أداة `flyctl` ثم `fly launch` و `fly deploy`.

في كل الحالات لا تحتاج تعديل الكود، فقط تأكد أن الأمر `npm start` يعمل، والمنصة ستتكفل بالباقي.
السيرفر مكتوب ليقرأ رقم المنفذ (Port) تلقائياً من متغير البيئة `PORT`، وهو ما تتطلبه كل هذه المنصات.

## طريقة اللعب داخل التطبيق

1. افتح الرابط → اكتب اسمك → اضغط **"إنشاء غرفة جديدة"**.
2. في غرفة الانتظار اضغط **"نسخ الرابط"** وأرسله لأصدقائك الثلاثة عبر واتساب مثلاً.
3. كل صديق يفتح الرابط، يكتب اسمه، ويضغط **"انضمام إلى غرفة"** (الكود يُملأ تلقائياً من الرابط).
4. بمجرد انضمام لاعبين اثنين على الأقل، يضغط المضيف (👑) زر **"بدء اللعبة"**.
5. تحكم بقطعتك عبر الأسهم الظاهرة أسفل الشاشة، أو لوحة المفاتيح، أو السحب بإصبعك على اللوحة.
6. تجنب الاصطدام بالحوائط وبأي أثر ملوّن (حتى لونك). آخر لاعب ينجو أو صاحب أكبر مساحة يفوز.
7. بعد النهاية يمكن للمضيف الضغط على **"إعادة اللعب"** ليعود الجميع لغرفة الانتظار من جديد.

## هيكل المشروع

```
tron-game/
  server.js          # منطق اللعبة الكامل + إدارة الغرف (Node.js + Socket.IO)
  package.json
  public/
    index.html        # الشاشات: الرئيسية، الانتظار، اللعب، النهاية
    style.css
    client.js          # الرسم على Canvas + التحكم + التواصل مع السيرفر
```

يمكنك تعديل حجم اللوحة (`COLS`, `ROWS`) أو سرعة الحركة (`TICK_MS`) من أعلى ملف `server.js`.
